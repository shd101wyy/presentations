Shor's Algorithm

Shor’s algorithm is an integer factorization algorithm designed by Perter Shor in 1994.
The fastest known classical factoring algorithm, the general number sieve works in subexponential time, specifically O(elogN(loglogN)2/3)O(e^{logN}(log log N)^{2/3})O(elogN(loglogN)2/3).
Shor’s algorithm achieves the same task in O((logN)3)O((logN)^3)O((logN)3)

Wikipedia

This algorithm finds the prime factorization of an n-bit integer in O(n3)O(n^3)O(n3) time whereas the best known classical algorithm requires 2O(n1/3)2^{O(n^{1/3})}2O(n1/3) time and the best upper bound for the complexity of this problem is O(2n/3+o(1))O(2^{n/3+o(1)})O(2n/3+o(1)).

How we use Shor's algorithm to break RSA

RSA

  1. We pick two prime number ppp and qqq
    Lets take p=3p=3p=3 and q=11q=11q=11

  2. We compute value of nnn and ϕ\phiϕ
    n=p∗qn = p * qn=p∗q and ϕ=(p−1)×(q−1)\phi = (p-1)\times(q-1)ϕ=(p−1)×(q−1)

Here we have

n=3×11=33n = 3 \times 11 = 33n=3×11=33
ϕ=(3−1)×(11−1)=2×10=20\phi = (3-1)\times(11-1) = 2 \times 10 = 20ϕ=(3−1)×(11−1)=2×10=20

  1. Find the value of eee (public key)
    Choose eee such that eee should be co-prime with ϕ\phiϕ. (Co-prime means it should not mulitiply by factors of ϕ\phiϕ and also not divide by ϕ\phiϕ)

Factors of ϕ\phiϕ are, 20=5×4=5×2×220 = 5 \times 4 = 5 \times 2 \times 220=5×4=5×2×2 so eee should not mutiply by 555 and 222 and shouldn't divid by 202020

So primes are 3, 7, 11, 17, 19 ..., as 3 and 11 are taken, we pick 7 as e for example.

e=7e = 7e=7

  1. Compute the value of ddd (private key)

e×d=1 mod ϕe \times d = 1\text{ }mod\text{ }\phie×d=1 mod ϕ

In our case we have p=3p=3p=3 q=11q=11q=11 n=3×11=33n=3\times11=33n=3×11=33 ϕ=(3−1)(11−1)=20\phi=(3-1)(11-1)=20ϕ=(3−1)(11−1)=20 e=7e=7e=7
We get d=3d=3d=3

  1. Encryption and Descryption

Encrypting a message mmm (number) with the public key (n,e)(n, e)(n,e) is calcualted:

m′=me(mod n)m' = m^{e} (mod\text{ }n)m′=me(mod n)

Decrypting with the private key (n,d)(n,d)(n,d) is done analogously with

m′′=m′d(mod n)m'' = m'^d(mod\text{ }n)m′′=m′d(mod n)
This is m′′=me×d(mod n)m''=m^{e \times d}(mod \text{ } n)m′′=me×d(mod n)

RSA now exploits the property that xa=xb(mod n)x^a = x^b(mod \text{ } n)xa=xb(mod n) if a=b(mod ϕ)a = b (mod\text{ }\phi)a=b(mod ϕ) As eee and ddd were chosen appropriately, it is m′′=mm'' = mm′′=m

So in order to crack the private key ddd, one needs to know ϕ=(p−1)(q−1)\phi = (p-1)(q-1)ϕ=(p−1)(q−1).
This in turn requires factoring n=p×qn = p \times qn=p×q, for which there is no known polynomial time algorithm.

Here comes the Shor's algorithm, which is a quantum computer algorithm for integer factorization that runs in polynormial time.

Traditional algorithm might take 1 million years to factor a number, will Shor's algorithm only takes 1 second.
